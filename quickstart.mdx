Dev Setup gets a new Mac from zero to fully configured in one short session. The process breaks into two phases: preparing macOS (updates + Xcode tools), then running whichever setup modules you need. The steps below walk you through the fastest path to a working environment.

<Steps>
  <Step title="Check prerequisites">
    Before you begin, confirm you have:

    - **macOS 10.9 Mavericks or later** (scripts were tested on Yosemite and El Capitan)
    - **Git** — available once Xcode Command Line Tools are installed (step 3 handles this automatically if needed)
    - **Admin privileges** — the scripts use `sudo` at several points; you must be an administrator on the machine
  </Step>
  <Step title="Clone the repository">
    Clone Dev Setup to your local machine and move into the project directory.

    ```bash
    git clone https://github.com/donnemartin/dev-setup.git && cd dev-setup
    ```

    <Tip>
      No Git yet? You can also bootstrap without it by downloading the `.dots` script directly and passing your chosen arguments:

      ```bash
      curl -O https://raw.githubusercontent.com/donnemartin/dev-setup/master/.dots && ./.dots [args]
      ```
    </Tip>
  </Step>
  <Step title="Run osxprep.sh to update macOS and install Xcode tools">
    Run the preparation script first. It installs all pending macOS software updates and then installs the **Xcode Command Line Tools** — a required dependency for Homebrew and many other tools.

    ```bash
    ./osxprep.sh
    ```

    A dialog box will appear asking you to confirm the Xcode Command Line Tools installation. Click **Install** and wait for it to complete, then follow any remaining terminal prompts.

    <Note>
      If macOS updates require a restart, reboot your machine and run `./osxprep.sh` again. It will pick up where it left off and proceed to install the Xcode tools.
    </Note>
  </Step>
  <Step title="Run the .dots launcher">
    With your machine up to date and Xcode tools installed, run `.dots` with the modules you want. Pass `all` to install every available script, or name individual modules to install only what you need.

    **Install everything:**

    ```bash
    ./.dots all
    ```

    **Install core tools only** (`bootstrap`, `brew`, `osx`):

    ```bash
    ./.dots bootstrap brew osx
    ```

    **Install core tools plus the Python data stack:**

    ```bash
    ./.dots bootstrap brew osx pydata
    ```

    **Install core tools plus data stores and AWS:**

    ```bash
    ./.dots bootstrap osxprep brew osx pydata aws datastores
    ```

    <Note>
      `.dots` will prompt you before it starts: `"This script may overwrite existing files in your home directory. Are you sure? (y/n)"`. Type `y` to proceed. The script then requests your administrator password and keeps it alive for the duration of the run — you won't need to re-enter it repeatedly.
    </Note>

    When the script finishes, you'll see:

    ```
    ------------------------------
    Completed running .dots, restart your computer to ensure all updates take effect
    ------------------------------
    ```

    **Restart your Mac** to apply all changes.
  </Step>
</Steps>

## Available modules

Pass any combination of these names to `.dots`:

| Argument | Script | What it does |
|---|---|---|
| `bootstrap` | `bootstrap.sh` | Syncs dotfiles to your home directory `~` |
| `osxprep` | `osxprep.sh` | Updates macOS and installs Xcode Command Line Tools |
| `brew` | `brew.sh` | Installs Homebrew, formulae, and GUI apps |
| `osx` | `osx.sh` | Applies developer macOS defaults |
| `pydata` | `pydata.sh` | Sets up Python + data science packages |
| `aws` | `aws.sh` | Installs AWS CLI, Boto, Spark, MapReduce |
| `datastores` | `datastores.sh` | Installs MySQL, PostgreSQL, MongoDB, Redis, Elasticsearch |
| `web` | `web.sh` | Installs Node.js, JSHint, Less |
| `android` | `android.sh` | Installs Java, Android SDK, Android Studio |
| `all` | *(all of the above)* | Runs every script in sequence |
