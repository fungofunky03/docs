Dev Setup provides a `.gitconfig`, a global `.gitignore`, and a `.gitattributes` file that work together to give you a productive, colorful Git experience out of the box. The configuration covers color output, a rich set of shorthand aliases, sensible merge and push defaults, and automatic exclusion of macOS and IDE artifacts from every repository on your machine.

## Setting Your Identity

The `.gitconfig` shipped by Dev Setup intentionally omits `user.name` and `user.email` — those are personal and should never land in a shared repository. Set them once after running `bootstrap.sh`:

```bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
```

<Note>
  The recommended place for your Git identity (and a GitHub token if you use one) is `~/.extra`, which is sourced by `.bash_profile` but never committed or overwritten by `bootstrap.sh`:

  ```bash
  # ~/.extra
  GIT_AUTHOR_NAME="Your Name"
  GIT_COMMITTER_NAME="$GIT_AUTHOR_NAME"
  git config --global user.name "$GIT_AUTHOR_NAME"

  GIT_AUTHOR_EMAIL="you@example.com"
  GIT_COMMITTER_EMAIL="$GIT_AUTHOR_EMAIL"
  git config --global user.email "$GIT_AUTHOR_EMAIL"
  ```
</Note>

To cache HTTPS credentials in the macOS Keychain so you aren't prompted on every push:

```bash
git config --global credential.helper osxkeychain
```

## Core Settings

```ini
[apply]
    # Detect whitespace errors when applying a patch
    whitespace = fix

[core]
    # Use custom .gitignore and .gitattributes
    excludesfile = ~/.gitignore
    attributesfile = ~/.gitattributes

    # Treat spaces before tabs and trailing whitespace as errors
    whitespace = space-before-tab,-indent-with-non-tab,trailing-space

    # Make `git rebase` safer on OS X
    trustctime = false

    # Prevent showing files with non-ASCII names as unversioned
    precomposeunicode = false

[diff]
    # Detect copies as well as renames
    renames = copies

[help]
    # Automatically correct and execute mistyped commands
    autocorrect = 1

[merge]
    # Include summaries of merged commits in merge commit messages
    log = true

[push]
    default = matching
    # Push relevant annotated tags when pushing branches
    followTags = true
```

## Color Settings

Dev Setup enables color output for all capable Git commands and assigns specific colors to each state indicator:

```ini
[color]
    ui = auto

[color "branch"]
    current = yellow reverse
    local = yellow
    remote = green

[color "diff"]
    meta = yellow bold
    frag = magenta bold   # line info
    old = red             # deletions
    new = green           # additions

[color "status"]
    added = yellow
    changed = green
    untracked = cyan
```

## Git Aliases

Dev Setup defines a comprehensive set of aliases that cover the most common daily workflows. Use them with the full `git` command or with the `g` alias defined in `.aliases`.

<Accordion title="View all configured aliases">

| Alias | Expands to | Description |
|-------|-----------|-------------|
| `l` | `log --pretty=oneline -n 20 --graph --abbrev-commit` | Compact graph of the last 20 commits |
| `s` | `status -s` | Short-format working tree status |
| `d` | `diff-index --quiet HEAD \|\| clear; diff --patch-with-stat` | Diff between HEAD and current state |
| `di` | `diff --patch-with-stat HEAD~$1` | Diff `$n` revisions back (e.g. `git di 3`) |
| `p` | `git pull; git submodule foreach git pull origin master` | Pull repo and all submodules |
| `c` | `clone --recursive` | Clone including all submodules |
| `ca` | `git add -A && git commit -av` | Stage everything and commit verbosely |
| `go` | `checkout -b "$1" 2>/dev/null \|\| checkout "$1"` | Switch to branch, creating it if needed |
| `tags` | `tag -l` | List all tags |
| `branches` | `branch -a` | List all branches (local + remote) |
| `remotes` | `remote -v` | Show all remotes with URLs |
| `amend` | `commit --amend --reuse-message=HEAD` | Amend the latest commit without editing the message |
| `credit` | `commit --amend --author "$1 <$2>" -C HEAD` | Credit a co-author on the latest commit |
| `reb` | `rebase -i HEAD~$1` | Interactive rebase of the last `$n` commits |
| `retag` | Delete old tag, push removal, re-tag latest commit | Re-tag the latest commit with an existing tag name |
| `fb` | `branch -a --contains $1` | Find branches containing a given commit |
| `ft` | `describe --always --contains $1` | Find tags containing a given commit |
| `fc` | `log ... -S$1` | Find commits that introduced or removed a string |
| `fm` | `log ... --grep=$1` | Find commits by message substring |
| `dm` | `branch --merged \| grep -v '*' \| xargs -n1 git branch -d` | Delete all branches already merged into the current branch |
| `contributors` | `shortlog --summary --numbered` | List contributors by commit count |
| `mpr` | Fetch, rebase, merge, and close a GitHub pull request | Merge a GitHub PR on top of `master` |

</Accordion>

### Commonly Used Aliases

```ini
[alias]
    # Compact graph log of the last 20 commits
    l = log --pretty=oneline -n 20 --graph --abbrev-commit

    # Short status
    s = status -s

    # Stage everything and commit (verbose)
    ca = !git add -A && git commit -av

    # Switch to a branch, creating it if necessary
    go = "!f() { git checkout -b \"$1\" 2> /dev/null || git checkout \"$1\"; }; f"

    # Amend the latest commit without changing its message
    amend = commit --amend --reuse-message=HEAD

    # Delete branches already merged into the current branch
    dm = "!git branch --merged | grep -v '\\*' | xargs -n 1 git branch -d"

    # Interactive rebase of the last N commits
    reb = "!r() { git rebase -i HEAD~$1; }; r"

    # List contributors with commit counts
    contributors = shortlog --summary --numbered
```

## URL Shorthands

The `.gitconfig` includes URL rewrite rules so you can use short `gh:` and `gist:` prefixes when cloning or pushing:

```ini
[url "git@github.com:"]
    insteadOf = "gh:"
    pushInsteadOf = "github:"
    pushInsteadOf = "git://github.com/"

[url "git://github.com/"]
    insteadOf = "github:"

[url "git@gist.github.com:"]
    insteadOf = "gst:"
    pushInsteadOf = "gist:"
    pushInsteadOf = "git://gist.github.com/"
```

For example, `git clone gh:donnemartin/dev-setup` is equivalent to `git clone git@github.com:donnemartin/dev-setup`.

## Global .gitignore

The `.gitignore` file is registered as `core.excludesfile` so its patterns apply to every repository on your machine without touching project-level `.gitignore` files:

```gitignore
# IntelliJ IDEA IDE
.idea

# Compiled Python files
*.pyc

# Folder view configuration files
.DS_Store
Desktop.ini

# Thumbnail cache files
._*
Thumbs.db

# Files that might appear on external disks
.Spotlight-V100
.Trashes

# Files that contain credentials or personal info
.extra
.boto
.mrjob.conf
.s3cfg
.aws/

# Repo scratch directory
scratch/
```

<Warning>
  Credential files (`.boto`, `.mrjob.conf`, `.s3cfg`, `.aws/`) are excluded globally. Never override this behavior by adding them to a project-level `.gitignore` allow-list — your secrets must never enter version control.
</Warning>

## .gitattributes

The `.gitattributes` file is registered as `core.attributesfile`. It currently documents the automatic line-ending normalization rule but leaves it disabled due to a known upstream issue:

```gitattributes
# Automatically normalize line endings for all text-based files
#* text=auto
# Disabled because of https://github.com/mathiasbynens/dotfiles/issues/149
```

If you work exclusively on macOS or Linux and want consistent LF line endings across all text files, you can enable this rule by uncommenting the line in `~/.gitattributes`.
