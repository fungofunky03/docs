Dev Setup's Bash configuration is split across several focused files that `.bash_profile` assembles in a defined order. Each file has a single responsibility — prompt rendering, environment exports, aliases, or utility functions — so you can read, override, or replace individual pieces without touching the others.

## How the Files Chain Together

When you open a new terminal, macOS runs `.bash_profile` as a login shell. Dev Setup's `.bash_profile` sources each configuration file in sequence:

```bash
for file in ~/.{path,bash_prompt,exports,aliases,functions,extra}; do
    [ -r "$file" ] && [ -f "$file" ] && source "$file";
done;
```

| Load order | File | Role |
|-----------|------|------|
| 1 | `~/.path` | Extend `$PATH` before any feature detection (optional, never overwritten) |
| 2 | `~/.bash_prompt` | Define the `$PS1` prompt string |
| 3 | `~/.exports` | Set environment variables |
| 4 | `~/.aliases` | Define shell aliases |
| 5 | `~/.functions` | Define shell functions |
| 6 | `~/.extra` | Personal overrides and secrets (optional, never overwritten) |

`.bashrc` is a one-liner that delegates to `.bash_profile` for non-login interactive shells:

```bash
[ -n "$PS1" ] && source ~/.bash_profile;
```

## .bash_profile: Shell Options and Completions

Beyond sourcing the dotfile chain, `.bash_profile` sets several `shopt` options and registers tab-completion handlers:

```bash
# Case-insensitive globbing (used in pathname expansion)
shopt -s nocaseglob;

# Append to Bash history instead of overwriting it
shopt -s histappend;

# Autocorrect typos in path names when using `cd`
shopt -s cdspell;

# Enable Bash 4 features when possible
for option in autocd globstar; do
    shopt -s "$option" 2> /dev/null;
done;

# Enable tab completion for `g` (alias for git)
if type _git &> /dev/null && [ -f /usr/local/etc/bash_completion.d/git-completion.bash ]; then
    complete -o default -o nospace -F _git g;
fi;

# Add tab completion for SSH hostnames from ~/.ssh/config
[ -e "$HOME/.ssh/config" ] && complete -o "default" -o "nospace" \
  -W "$(grep "^Host" ~/.ssh/config | grep -v "[?*]" | cut -d " " -f2- | tr ' ' '\n')" scp sftp ssh;
```

## .bash_prompt: Custom Prompt with Git Status

The prompt is based on the Solarized Dark palette and uses `tput` for portable color codes. It shows your username, hostname, current directory, and — when inside a Git repository — the branch name with dirty-state indicators.

```
username at hostname in ~/projects/dev-setup on main [+!?$]
$
```

The status indicators appended to the branch name mean:

| Symbol | Condition |
|--------|-----------|
| `+` | Staged but uncommitted changes |
| `!` | Unstaged changes to tracked files |
| `?` | Untracked files |
| `$` | Stashed changes |

The hostname appears in **bold red** when connected via SSH, and the username appears in red when running as `root` — both are immediate visual warnings.

```bash
# Highlight the user name when logged in as root.
if [[ "${USER}" == "root" ]]; then
    userStyle="${red}";
else
    userStyle="${orange}";
fi;

# Highlight the hostname when connected via SSH.
if [[ "${SSH_TTY}" ]]; then
    hostStyle="${bold}${red}";
else
    hostStyle="${yellow}";
fi;
```

The secondary prompt (`PS2`) uses a yellow `→` arrow for continuation lines.

```bash
# Detect GNU vs BSD ls and pick the right color flag
if ls --color > /dev/null 2>&1; then
    colorflag="--color"   # GNU ls

alias l="ls -lF ${colorflag}"          # long format
```

## .exports: Environment Variables

```bash
# Make vim the default editor
export EDITOR='vim';

# Increase Bash history to 32,768 entries; omit duplicates and space-prefixed commands
export HISTSIZE='32768';
export HISTFILESIZE="${HISTSIZE}";
export HISTCONTROL='ignoreboth';

# Prefer US English and UTF-8
export LANG='en_US.UTF-8';
export LC_ALL='en_US.UTF-8';

# Highlight section titles in man pages
export LESS_TERMCAP_md="${yellow}";

# Don't clear the screen after quitting a man page
export MANPAGER='less -X';

# Always enable colored grep output
export GREP_OPTIONS='--color=auto';

# Persistent Node REPL history (32,768 entries)
NODE_REPL_HISTORY_FILE=~/.node_history;
NODE_REPL_HISTORY_SIZE='32768';
```

## .functions: Utility Shell Functions

Dev Setup includes a library of utility functions. Here are the most useful ones:

<Accordion title="mkd — mkdir and cd in one step">

```bash
function mkd() {
    mkdir -p "$@" && cd "$_";
}
```

Create a directory (including intermediate parents) and immediately `cd` into it.

</Accordion>

<Accordion title="fs — human-readable file/directory size">

```bash
function fs() {
    if du -b /dev/null > /dev/null 2>&1; then
        local arg=-sbh;
    else
        local arg=-sh;
    fi
    if [[ -n "$@" ]]; then
        du $arg -- "$@";
    else
        du $arg .[^.]* *;
    fi;
}
```

Run `fs` with no arguments to see the sizes of everything in the current directory, including hidden files.

</Accordion>

<Accordion title="targz — smart compressed archive creator">

```bash
function targz() {
    local tmpFile="${@%/}.tar";
    tar -cvf "${tmpFile}" --exclude=".DS_Store" "${@}" || return 1;
    # Picks zopfli > pigz > gzip depending on what's available and file size
    ...
    echo "${tmpFile}.gz created successfully.";
}
```

Automatically selects the best available compressor (`zopfli` for files under 50 MB, then `pigz`, then `gzip`).

</Accordion>

<Accordion title="server — instant Python HTTP server">

```bash
function server() {
    local port="${1:-8000}";
    sleep 1 && open "http://localhost:${port}/" &
    python -c $'import SimpleHTTPServer;\n...' "$port";
}
```

Serve the current directory over HTTP. Opens the browser automatically. Defaults to port 8000.

</Accordion>

<Accordion title="calc — command-line calculator">

```bash
function calc() {
    local result="";
    result="$(printf "scale=10;$*\n" | bc --mathlib | tr -d '\\\n')";
    # Cleans up decimal formatting: adds leading zeros, strips trailing zeros
    ...
    printf "\n";
}
```

Example: `calc 2^32`, `calc "sqrt(2)"`, `calc "1/3"`.

</Accordion>

<Accordion title="json — pretty-print and syntax-highlight JSON">

```bash
function json() {
    if [ -t 0 ]; then
        python -mjson.tool <<< "$*" | pygmentize -l javascript;
    else
        python -mjson.tool | pygmentize -l javascript;
    fi;
}
```

Works with an argument (`json '{"key":"val"}'`) or via pipe (`cat data.json | json`).

</Accordion>

<Accordion title="gz — compare original vs gzipped file size">

```bash
function gz() {
    local origsize=$(wc -c < "$1");
    local gzipsize=$(gzip -c "$1" | wc -c);
    local ratio=$(echo "$gzipsize * 100 / $origsize" | bc -l);
    printf "orig: %d bytes\n" "$origsize";
    printf "gzip: %d bytes (%2.2f%%)\n" "$gzipsize" "$ratio";
}
```

Useful for checking how well a file will compress before deploying it.

</Accordion>

<Accordion title="tre — enhanced tree view">

```bash
function tre() {
    tree -aC -I '.git|node_modules|bower_components' --dirsfirst "$@" | less -FRNX;
}
```

Shows a colorized directory tree, hides `.git`, `node_modules`, and `bower_components`, and pipes through `less` when the output is longer than your terminal height.

</Accordion>

## .inputrc: Readline Key Bindings

`.inputrc` configures the GNU Readline library, which drives input in Bash, Python's REPL, and other interactive tools:

```bash
# Case-insensitive tab completion
set completion-ignore-case on

# Show all matches when multiple completions exist
set show-all-if-ambiguous on

# Add a trailing slash when completing symlinks to directories
set mark-symlinked-directories on

# Up/Down arrows search history using the typed prefix
"\e[B": history-search-forward
"\e[A": history-search-backward

# Do not complete hidden files unless the pattern starts with a dot
set match-hidden-files off

# Show all completions at once (no paging)
set page-completions off

# Ask before showing more than 200 completions
set completion-query-items 200

# Show file type indicators (like ls -F) in completions
set visible-stats on

# Smart completion: don't re-type text that already appears after the cursor
set skip-completed-text on

# Allow UTF-8 input and output
set input-meta on
set output-meta on
set convert-meta off

# Alt+Delete deletes the preceding word
"\e[3;3~": kill-word
```

<Tip>
  Add your own aliases and exports to `~/.extra` rather than editing `.aliases` or `.exports` directly. Dev Setup's `bootstrap.sh` will overwrite those files on the next sync, but `~/.extra` is explicitly excluded from version control and never touched by the sync script.
</Tip>
